# 文件与数据

本页整理代码、数据、模型权重与输出文件在平台上的组织方式。

## 建议目录

```text
~
├── courses/
│   └── course-name/
│       ├── code/
│       ├── data/
│       ├── runs/
│       └── scripts/
└── envs-or-tools/
```

不同课程或实验可以自行调整，但建议把代码、输入数据、运行输出和作业脚本分开，便于排查和复现。

## 上传与下载

如果当前平台只能通过图形界面上传下载文件，建议采用“本地打包、平台上传、登录节点解包”的方式。

```bash
tar -czvf project.tar.gz project/
tar -xzvf project.tar.gz
```

## 大文件与数据集

- 数据集尽量按课程或项目共享约定存放，避免每个人重复上传大文件；
- 作业输出不要写到难以定位的临时目录；
- 训练日志、模型权重、图表和最终报告应分目录保存；
- 删除中间文件前先确认课程是否需要复现实验结果。

## 可复现记录

每个项目建议保留：

- 代码版本或提交号；
- 运行命令；
- 作业脚本；
- 环境文件，例如 `environment.yml` 或 `requirements.txt`；
- 关键输出日志。
