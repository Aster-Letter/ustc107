# 107 平台图形界面 (GUI) 使用指南

107 本科生算力平台为不熟悉 Linux 命令行、SSH 密钥配置，或者需要使用图形化科学计算软件（如 MATLAB GUI、JupyterLab、VSCode）的同学，提供了功能强大的 Web 图形化交互界面。通过浏览器，你即可无缝完成从文件传输、环境切换到任务提交、报错排查的完整科研/大作业全链路。

---

## 1. 登录图形界面与接入集群

访问平台主页 [107.ustc.edu.cn](https://107.ustc.edu.cn) 并通过统一身份认证（USTC CAS）登录后，点击顶部导航栏的 **【登录集群】**。


平台提供了两种无需本地客户端的免密接入标签页：

### 1.1 命令行 (Shell) 标签页
* **操作**：直接点击卡片右下角的 **【打开】** 按钮。
* **效果**：浏览器会自动弹出一个功能完整的 Web Shell 窗口（默认接入登录节点，如图中的 `tradmin-02`）。
* **适用场景**：适合随时随地进行快捷的命令行操作（如 `squeue` 查看作业、`git clone` 拉取代码、查看日志等）。

### 1.2 桌面 (Desktop) 标签页
* **操作**：切换至 **【桌面】** 标签页，点击右侧的 **【+ 新建桌面】**。系统会为你初始化一个可视化节点。
* **连接桌面**：创建完成后，点击卡片右下角的 **【连接】** 按钮：


* **可视化环境**：连接成功后，浏览器将在新标签页中唤起远程桌面。107 平台底层采用轻量高效的 [Xfce 桌面环境](https://www.xfce.org/)，并通过 [VNC (Virtual Network Computing) 协议](https://en.wikipedia.org/wiki/Virtual_Network_Computing)进行画面传输：


* **适用场景**：适合运行高度依赖 GUI 的科学计算或建模软件（如 MATLAB GUI、Mathematica、GROMACS 建模、大模型图片生成预览等）。

> ⚠️ **【红线警告】登录节点禁算规范**：
> 无论是通过网页版【Shell】还是【远程桌面 (VNC)】接入，你当前所处的环境都只是**登录节点**（如 `tradmin-02`）。登录节点由全校同学共享，**请切勿在登录节点直接运行高负载的重型计算或模型训练任务**（例如直接在 VNC 的终端里执行 `python train.py`）。这会导致节点卡死并影响全校同学正常登录。所有计算任务请务必通过下文所述的【应用中心】或【提交作业表单】提交至计算节点运行！

## 2. 查看集群资源使用情况

在提交大型计算任务前，养成先查看集群整体负载的习惯，可以帮你合情合理地预估排队时间。点击顶部导航栏的 **【仪表盘】**：

在仪表盘中，你可以直观监控以下四大核心数据：
* **节点 (Nodes)**：展示当前集群的总节点数及空闲节点比例。
* **CPU**：实时显示当前全校本科生占用的 CPU 核心总数与空闲核心数。
* **加速卡 (GPU)**：**这是深度学习任务最核心的指标。** 观察“空闲”卡数。如果空闲卡数为 0，意味着此时提交 GPU 作业必然需要排队。
* **作业**：实时显示当前全校正在 `运行中` 与 `排队中` 的作业总数。如果排队作业较多，建议避开高峰期或适当缩短自己申请的作业时长（Time Limit）以获得更高的调度优先级。

## 3. 网页端交互式应用中心 (JupyterLab / VSCode)

如果你不想编写硬核的 Slurm 批处理脚本，想直接在网页上边写代码、边画图、边调试，可以利用集成了容器技术的 **【应用中心】**box，一键式拉起在计算节点上运行的交互式开发环境。

点击顶部导航栏的 **【应用】** $\rightarrow$ **【创建应用】**：

平台目前内置了两种最主流的集成开发环境：
1.  **[JupyterLab](https://jupyterlab.readthedocs.io/en/stable/)**：适合数据分析、机器学习、深度学习及作业快速迭代。
2.  **VSCode ([Apptrainer / Singularity Container](https://apptrainer.org/))**：通过高性能科学计算容器技术隔离出来的网页版 VSCode 编辑器，支持安装丰富的插件。

### 3.1 详细资源配置表单指南
点击对应的应用图标后，会进入资源申请与配置页面。你需要严格按照任务需求填写表单，**乱填或多填会导致作业因超限被拦截或由于申请资源过多而陷入无限排队**：


**💡 核心字段填写说明：**
* **作业名**：系统会自动生成包含时间戳的默认名称，通常无需修改。
* **账户**：一般固定选择 `stu`（本科生公共账户）。
* **分区 (Partition)**：整个集群在物理或逻辑上划分为不同的区域。本科生统一选择 `Students` 分区。进一步了解可参考 [Slurm 分区概念官方文档](https://slurm.schedmd.com/documentation.html)。
* **QOS (服务质量 / Quality of Service)**：
  * 用于定义作业的优先级、资源上限和时间限制。通常选择 `qos_stu_default` 或 `qos_stu001`。关于调度权重可拓展阅读 [Slurm QOS 深入解析教程](https://slurm.schedmd.com/qos.html)。
  * *注：大作业高峰期，指导助教或平台可能会通知切换至特定的 QOS 组。*
* **节点数**：对于绝大多数单卡/单机任务，**请务必填 `1`**。
* **单节点加速卡数 (GPU 数量)**：
  * **深度学习任务**：填 `1`（即申请 1 张 GPU 显卡）。
  * **纯 CPU 计算任务**：**请务必填 `0`**！如果不跑深度学习却申请了 GPU，会导致宝贵的公共显卡资源被白白空置浪费，且会被系统后台标记。
* **最长运行时间**：根据作业长短预估（单位：分钟，如填 `60` 表示一小时）。超时后系统会执行强行回收，**请务必在代码中加入定期保存 Checkpoint/模型权重的逻辑**。
* **Conda环境名 (高频快捷功能)**：
  * 如果你在底层的 [Anaconda / Miniconda](https://docs.conda.io/en/latest/) 中自己配置了虚拟环境（例如名为 `dl_env`），可以直接在这里的输入框中填入对应的环境名。
  * 应用启动后，系统会自动帮你执行 `conda activate dl_env`，进入环境后即可直接 `import torch`，免去手动去终端激活的烦恼。想深入学习环境管理可看 [Conda 官方环境指南](https://docs.conda.io/projects/conda/en/latest/user-guide/tasks/manage-environments.html)。
* **其他sbatch参数**：用于额外限制核心数。例如填 `-c 4` 表示申请 4 个 CPU 核心，能够有效防止默认分配的核心数过大引发 QOS 拦截。

填写完成后点击 **【提交】**，在【已创建的应用】列表中等待状态从 `提交中` 变为 `运行中`，点击右侧弹出的 **【连接】** 按钮即可开始 coding！

## 4. 网页端作业提交表单 (Web Slurm)

除了通过交互式应用（Jupyter/VSCode）或纯命令行 `sbatch` 提交任务外，平台还贴心地提供了图形化的 **【提交作业】** 表单，让你可以免去手工编写底层集群作业调度系统的批处理脚本文件的烦恼。

107 平台底层基于国际主流的 **[Slurm (Simple Linux Utility for Resource Management) 集群资源调度系统](https://slurm.schedmd.com/)**。点击顶部导航栏的 **【作业】** $\rightarrow$ **【提交作业】**。

### ✍️ 如何通过表单提交批处理任务：
1.  **命令输入框**：在此处直接输入你原本需要写在脚本里的执行命令，例如：
    ```bash
    source /home/scc/你的学号/miniconda3/bin/activate myenv
    cd /home/scc/你的学号/project/
    python train.py --epochs 10 --batch_size 64
    ```
    *注：框中自带的 `#SBATCH` 注释仅作格式参考，网页端实际生效的资源配额**完全以你在下方表单里勾选填写的数值为准**。*
2.  **资源配额勾选**：在下方级联菜单中，选定你的账户（`stu`）、分区（`Students`）、QOS（`qos_stu001`）、节点数（`1`）以及加速卡数。
3.  **工作目录**：点击 **【选择】** 按钮，在弹出的目录树中，精确指向你的代码和数据集所在的绝对路径。
4.  **标准输出/错误文件**：定义日志输出文件的名称和存储路径（默认在当前工作目录下生成 `job.%j.out`，其中 `%j` 会被自动替换为 Slurm 系统实际分配的作业 ID）。想了解更多占位符转义可看 [Slurm 提交作业输出命名规范](https://slurm.schedmd.com/sbatch.html)。
5.  **一键提交**：确认无误后点击左下角红色的 **【提交】** 按钮。任务随即进入 Slurm 统一调度队列。

## 5. 作业管理与运行日志追踪

作业提交后（无论是通过网页表单提交、通过交互式应用创建，还是在本地用命令行 `sbatch` 提交的），都可以在 **【作业】** 模块中进行全生命周期管理。进一步学习 Slurm 状态机逻辑可阅读 [Slurm 作业状态官方词条解释](https://slurm.schedmd.com/squeue.html#SECTION_JOB-STATE-CODES)。

点击顶部导航栏的 **【作业】** $\rightarrow$ **【未结束的作业】**：
、

### 5.1 状态监控与快捷释放
* **多维筛选**：当大作业临近截止日期时，列表可能会被刷屏。你可以通过上方的 `作业ID` 搜索框快速捞出自己的任务。
* **状态识别**：
  * `PD` (Pending)：正在排队，等待集群空出资源。
  * `R` (Running)：正在计算节点正常运行。
* **快捷终止**：若通过查看日志（见下文）发现程序因代码写错发生死循环，或者刚跑两步就疯狂报错爆显存，**请立刻在最右侧的【操作】栏中点击取消/终止**，及时释放算力，避免扣减自己的机时额度。

### 5.2 网页端查看运行日志
点击对应任务的 **【作业ID】** 数字，会弹出一个详情面板。在面板中你可以直接查看实时刷新的标准输出日志（`stdout` / `stderr`）。**无需登录终端去执行繁琐的 `tail -f xxx.out` 命令**，方便快速观察训练的 Loss 下降情况或排查 Traceback 崩溃信息。

## 6. 高效便捷的网页文件管理

点击顶部导航栏的 **【文件管理】**，可以完全免去在本地安装配置 WinSCP、FileZilla 或 Mobaxterm Xftp 的繁琐步骤，直接在浏览器中像使用网盘一样管理你在集群上的个人主目录（`/home/scc/你的学号`）。


### 💡 高频操作技巧：
* **文件极速传输**：支持直接将本地的代码、权重、PDF 文档拖拽到浏览器窗口中进行上传；勾选计算好的结果文件/图表后，点击 **【下载】** 即可保存至本地。
* **高效打包与解压（核心避坑）**：
  * ⚠️ **请勿直接上传包含几万张图片的未压缩数据集文件夹！** 浏览器传输海量小文件会导致网络连接高频握手，速度极慢且容易丢包挂掉。
  * **正确做法**：在本地将数据集打包成一个单独的 `.zip` 或 `.tar.gz` 压缩包，通过网页端上传该单文件，上传完毕后勾选它，直接点击工具栏的 **【解压缩】**。集群底层会以极高的本地 I/O 速度瞬间完成解压。
* **在线代码编辑**：在列表里直接双击一个文本文件（如 `.py`、`.sh`、`.txt`），网页会自动弹出一个轻量级在线编辑器。你可以在这里直接修改代码参数并保存，免去了“本地改完再重复上传”的折腾。
* **一键切至当前路径终端**：点击右上角的 **【在 shell 中打开】**，可以一键在当前文件目录下唤起网页版 Shell，路径自动同步，非常贴心。
