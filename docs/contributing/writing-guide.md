# 写作指南

本文档用于统一本科生算力平台 wiki 的内容风格和组织方式。

## 读者优先级

优先服务以下读者：

1. 第一次使用平台的本科生；
2. 正在完成课程作业、实验或竞赛任务的同学；
3. 需要把课程材料迁移到平台上的助教和课程维护者；
4. 负责维护平台文档的同学。

## 页面类型

文档建议分为四类：

| 类型 | 放置位置 | 写作目标 |
| --- | --- | --- |
| 平台说明 | `overview/` | 解释平台是什么、能做什么、不能做什么 |
| 基础操作 | `basics/` | 提供跨课程复用的登录、环境、文件、作业、排障说明 |
| 学科用例 | `guides/<domain>/` | 按课程或任务给出从准备到运行的完整路径 |
| 维护说明 | `contributing/` | 约定文档写法、模板和仓库维护方式 |

## 推荐页面结构

用例页尽量按这个顺序写：

1. 适用对象；
2. 你将完成什么；
3. 前置条件；
4. 资源建议；
5. 环境准备；
6. 文件准备；
7. 作业脚本或 GUI 操作；
8. 提交与监控；
9. 输出结果检查；
10. 常见错误；
11. 进一步阅读。

## 写作规则

- 每页只解决一类问题；
- 标题直接描述任务，不使用含糊口号；
- 命令块前说明运行位置，例如本地、登录节点或计算节点；
- 所有长任务都给出最小测试命令；
- 所有资源参数都说明适用场景，不把示例值写成唯一正确值；
- 学科页面尽量链接到基础页面，避免重复解释 Slurm、Conda、文件上传等通用知识；
- 对暂不确定的信息使用 `!!! note` 或 `!!! warning` 标出，不要混在正文里。

## 从哪里开始写

建议先完成一条可跑通的最小链路：

1. `quickstart.md`：第一次登录到提交最小作业；
2. `basics/jobs.md`：批处理和交互式任务；
3. `basics/environments.md`：Miniforge/Mamba 与课程环境；
4. `guides/ai/deep-learning-homework.md`：把现有 AI 用例补成完整示范；
5. 再按课程需求扩展数学、物理化学和其他专业用例。
