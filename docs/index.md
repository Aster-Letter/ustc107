# USTC 本科生算力平台

当前文档版本：`v0.1` 测试版。

本科生算力平台为课程作业、训练项目、数值计算、统计模拟和科研训练等任务提供共享计算资源。你可以把它理解为一组由学校统一管理的服务器：你把代码、数据和作业要求提交上去，平台在合适的计算节点上运行任务，并把日志和结果留在你的工作目录中。

平台正式入口为 <https://107.ustc.edu.cn/>。如果课程群、助教或平台公告给出了新入口，请以最新通知为准。

!!! warning "临时性说明"
    `v0.1` 仍处于测试和迭代阶段，内容以帮助同学尽快完成课程和训练任务为主。平台后续可能进行迁移或架构重构，页面入口、资源字段、默认配额和调度策略都可能变化；遇到与平台页面不一致的地方，以平台页面和管理员说明为准。

如果你第一次接触命令行、Linux 或 Slurm，不需要先把所有概念学完。建议先按[快速开始](quickstart.md)完成一次最小闭环：进入平台、准备文件、提交一个测试任务、查看日志和结果。

## 你需要先知道

- 平台资源是共享的。不要在登录节点运行重计算任务，也不要长时间占用不用的交互式资源。
- GUI 是低门槛入口，适合第一次登录、上传下载文件、查看任务状态；命令行和作业脚本更适合可复现的训练、仿真和批量实验。
- 平台事实会变化，包括入口地址、资源配额、队列策略、GPU 型号、SSH 状态和预装软件。遇到页面中的占位参数时，请以平台页面或课程通知为准。
- 每一步都应该留下可检查的结果，例如日志文件、输出目录、作业 ID 和环境配置文件。

## 推荐阅读路径

第一次使用：

1. 读[快速开始](quickstart.md)，完成一次最小测试任务。
2. 读[GUI 使用](basics/gui.md)，知道在哪里上传文件、查看任务和下载结果。
3. 读[提交任务](basics/jobs.md)，理解交互式任务和批处理任务的区别。
4. 遇到报错时查[常见问题](basics/faq.md)。

准备长期项目：

1. 读[平台概览](overview/index.md)，理解登录节点、计算节点和共享存储。
2. 读[文件与数据](basics/files.md)，规划代码、数据、日志和结果目录。
3. 读[环境配置](basics/environments.md)，为项目建立独立环境。
4. 读[Slurm 速查](basics/slurm.md)，把实验流程写成可复现脚本。

## 场景指南

场景指南会把基础能力串成真实任务流程。目前已有[深度学习作业训练任务](guides/ai/deep-learning-homework.md)草稿；数学统计、物理化学和其他通用计算场景正在整理。

## 求助前准备

遇到问题时，建议先保存这些信息：

- 你在哪一步遇到问题。
- 作业 ID、提交脚本、运行命令。
- 日志文件中的完整错误信息。
- 你申请的 CPU、GPU、内存和运行时间。
- 代码、数据和输出文件所在路径，注意遮挡个人信息。
